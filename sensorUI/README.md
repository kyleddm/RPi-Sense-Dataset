This is a UI to see the data coming in in Real-Time.  Note:  This expects specific ID numbers to be in specific locations in the building where the data was recorded.  As such, I would recommend modifying this if you pull it down to re-use.  It requires tkinter, mqtt, and pygubu.  This UI used 10 Raspberry Pi devices with the SenseHat attachment.  IDs were randomly generated to split the 40 possible sensors into 17 sensor "nodes" of varying degree (1-4).  As a result, the output of the sensors (out.json files) use IDs from 1-17.  The corresponding parseData.py script converts these json files into a pandas dataframe that is sorted by RPi.  In addition, this UI splits the data visually by location (which matches the parsing done by parseData.py).
